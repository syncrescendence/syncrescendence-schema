## The CLI advantage that changes the calculus

The pre-grammar research was conducted through chat-surface Grok sessions where the operator manually injected context, steered the council, and iteratively refined prompts. That context is gone. The grammar layer now encodes what that context was groping toward — axis sets, epistemic requirements, convergence criteria, validation schemas.

The CLI fleet can receive the grammar layer as direct context injection:

- **Grok 4.1 Fast** has a 2M token context window at $0.20/M input. The entire grammar layer (~9 documents), the target modality class's axis set, the scaffold grammar's twelve-check schema, and the specific research question fit in a single prompt. Cost per research session: negligible.
- **Codex** can run grammar-injected prompts in parallel across multiple surfaces, with Adjudicator-grade meticulousness.
- **Gemini CLI** can run the same prompt with a different cognitive signature (Cartographer — structural homology detection), producing genuinely divergent decompositions rather than three versions of the same exegesis.
- **Commander** (Claude Code) can orchestrate, validate outputs against the twelve-check schema, and produce the synthesis.

The context-injection problem may be solved by the grammar layer itself becoming the prompt. The lost conversational context from the original Grok sessions encoded implicit knowledge about what to look for. The grammar layer encodes that knowledge explicitly, as typed axes with named primitives, epistemic labels, and validation criteria. If that's true, grammar-aligned CLI research should produce higher-fidelity evidence than the original chat-mediated research, because the decomposition targets are explicit rather than emergent.

## What Vizier needs from Vanguard

**Output mode:** Operational methodology design. Not diagnosis (we have that). Not schema (you've provided that). A concrete methodology that can be encoded as a CLI skill and executed against real surfaces.

### Ask 1 — The research prompt architecture

Design the structure of a grammar-aligned research prompt. Not one prompt — the template. What goes in it? In what order? What does the grammar layer contribute as context? What does the target surface contribute? What does the agent need to know about the axis set, the epistemic-labeling protocol, the convergence criteria, and the twelve-check validation to produce scaffold-ready evidence in a single pass?

Consider that different CLI agents have different context windows and cognitive signatures. Should Commander, Adjudicator, Cartographer, and Oracle all receive the same prompt, or should each receive a signature-adapted variant that plays to its strengths? If adapted, what varies and what stays constant?

### Ask 2 — The calibration protocol

The first grammar-aligned research cycle should target a surface that already has a pre-grammar scaffold (e.g., Slack — richest existing dossier). The delta between the grammar-aligned output and the existing scaffold reveals what pre-grammar research missed. Design the calibration protocol: what comparison structure reveals the delta most legibly? What metrics distinguish "the pre-grammar scaffold was mostly adequate" from "the pre-grammar scaffold has systematic blind spots"? How many calibration surfaces are needed before the methodology can be trusted on unknown surfaces?

### Ask 3 — The convergence methodology

The convergence question — how many surfaces before new convergent primitives stop appearing — was never asked during Phase 2. Design the methodology for answering it. This is a sampling theory problem applied to the tool universe. Given a modality class with potentially hundreds of surfaces, what sampling strategy determines when the convergent primitive set has stabilized? Is it sequential (keep adding surfaces until N consecutive additions produce zero new convergent primitives)? Statistical? Information-theoretic? What does Vanguard's strategic sense say about the right frame?

### Ask 4 — The disagreement extraction protocol

Your previous review prescribed: "The three agents should not be asked to converge first. They should be asked to expose contradictions, omissions, and differing classifications. Consensus is cheap. Useful disagreement is gold." Design the disagreement extraction protocol. When three agents produce decompositions of the same surface along the same axes, what comparison structure makes contradictions, omissions, and classification conflicts maximally visible? How does the synthesis agent (Commander) distinguish genuine disagreement (signaling ambiguity in the surface) from model noise (signaling nothing)?

### Ask 5 — The skill encoding

The end state is a CLI skill (SKILL.md) that any grammar-aligned research agent can invoke. What does that skill contain? What are its inputs, outputs, validation criteria, and failure modes? What does the operator need to provide (the research brief), and what does the skill provide (the decomposition frame, the epistemic requirements, the output schema)?
