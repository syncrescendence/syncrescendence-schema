# CONSTRUCTION BRIEF: KNOWLEDGE-ARCHITECTURE-ROOT.md

## Document Identity
**Name:** KNOWLEDGE-ARCHITECTURE-ROOT.md
**Artifact class:** Canon
**Teleology:** The mereological anchor of the entire knowledge system. Functions simultaneously as abstract, executive precis, table of contents, and constitutional root. Every other artifact in the system addresses from this document. Every gap in the system is visible from this document. Every future artifact knows where it fits before it is written.

## Scope Boundary
This document defines the architecture of the knowledge system itself — not the architecture of the institution's operations. It answers: how is knowledge organized, classified, promoted, maintained, and traversed? It does not answer: what should each surface do, how should agents be routed, or what the operational control plane looks like. Those belong in downstream artifacts (teleological manuals, specifications, registries).

The root document can and should declare open slots where the operational architecture is not yet mature. Open slots are features: they make incompleteness legible rather than hidden.
