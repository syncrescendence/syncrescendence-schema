## I. How the Grammar Was Made — The Production Pattern

The nine grammar-layer documents were produced across approximately 45 substantive turns spanning three threads (Schema-1: ~12 turns, Schema-2: ~17 turns, Schema-3 bridging: ~16 turns of Phase 2.5). The production pattern was consistent:

**Turn pattern per document:** Pre-build memo or construction brief (1 turn) → Draft with full context injection (1-2 turns) → Ratification review with bounded scope (1 turn) → Targeted correction if needed (0-1 turns) → Closure and state update (1 turn). Average: 3-4 turns per constitutional document.

**The dialogue dynamic that produced quality:** The Sovereign's intuition identified the pressure point. Vizier's analytical precision decomposed it into structural components. Vanguard's strategic lens validated the decomposition and caught blind spots. Oracle's adversarial sensing pressure-tested consensus. The quality emerged from the tension between these cognitive signatures, not from any single one.

**What made the grammar documents prescient:** They described the system's structural physics — what must be true for any knowledge architecture to hold together — rather than prescribing operational procedures. Descriptive before prescriptive. Substrate before runtime. The grammar doesn't tell you how to run a research cycle; it tells you what a research cycle must produce to be valid, what checks it must pass, and how its output earns authority. This descriptive-first discipline is why the grammar survived contact with the operational reality of Phase 2.5 — the orchestration pattern, the fleet architecture, the 73-release avalanche all changed the operational surface without invalidating the constitutional substrate.

### Research Program — 32 Questions in 6 Waves

**Wave 1 — Lane B: Operational Contracts (6 questions)**

These are the heart of Phase 3. The contracts must be extracted from real-world patterns, not invented.

1. What typed contract formats exist in production multi-agent systems for commissioning work between agents? Examine Symphony's WORKFLOW.md, OpenAI's Agents SDK Turn/Item/Thread primitives, Google A2A's task negotiation protocol, CrewAI's task assignment schema, and LangGraph's state-passing conventions. What are the minimal required fields for a machine-parseable research commission?

2. What receipt/acknowledgment patterns exist in production agent orchestration? How does Symphony confirm task completion? How do A2A agents report status? How does Hermes Agent's ACP adapter structure result messages? What must a receipt contain to be auditable (who ran, what authority, what input, what output, pass/fail)?

3. How do production systems handle artifact promotion (draft → reviewed → canonical)? Examine Git-based approval flows, PR review gates, Symphony's security model, and the AGENTS.md standard's compliance patterns. What governance checks exist between an agent producing an artifact and that artifact becoming authoritative?

4. What present-state update patterns exist for systems with a singular source of truth? How do distributed systems handle the single-writer problem for critical state files? Examine etcd's leader election, Kubernetes' API server reconciliation, and any agent-orchestration system that maintains a "current state" artifact.

5. What typed contract schemas are emerging from the AAIF ecosystem (MCP, ACP, AGENTS.md)? Are there formal schema definitions for inter-agent communication beyond the protocol specs? What does a well-typed agent-to-agent message look like in ACP v0.3?

6. How do existing multi-agent systems handle contract violations — when an agent produces output that doesn't match the commissioned format? What error-handling, retry, and escalation patterns exist?

**Wave 2 — Lane C: Coordination and Choreography (6 questions)**

7. What relay/dispatch patterns are used in production multi-agent coding systems? Compare centralized (Symphony, gstack, Composio), decentralized (Ruflo swarm, DeerFlow), and hybrid (MCO tech-lead pattern) architectures. What primitives does each use for task routing?

8. How do multi-agent systems handle concurrency when multiple agents want to write near the same state? Examine git worktree isolation (Vibe Kanban, Emdash, Claude Code worktrees), database-level isolation, mailbox coordination (Overstory's SQLite WAL), and merge-queue patterns.

9. What review-lane patterns exist for distinguishing different types of agent work? How does gstack route CEO review vs engineering review vs design review? How does Symphony distinguish security-sensitive from routine tasks? What queue/priority patterns apply?

10. How do production systems implement heartbeat/health monitoring for long-running agent tasks? Examine Paperclip's heartbeat scheduling, Claude Code's /loop 7-day monitoring, Hermes Agent's cron scheduling with platform delivery, and OpenClaw's gateway health checks.

11. What coordination patterns exist for the specific topology where one orchestrator (Commander) dispatches to heterogeneous CLI agents (codex, gemini-cli, grok-cli, hermes-agent) and synthesizes their returns? Is MCO the closest existing implementation? What does it do that raw bash scripting doesn't?

12. How do existing systems handle degraded-mode operation when one surface in a multi-surface architecture becomes unavailable? What fallback, retry, and rerouting patterns exist?

**Wave 3 — Lane D: Observability (5 questions)**

13. What observability standards exist for multi-agent AI systems as of March 2026? Examine OpenTelemetry extensions for LLM/agent workloads, Langfuse's trace model, Braintrust's evaluation framework, Helicone's monitoring approach, and Phoenix's observability model. What is the minimal viable observability stack?

14. What metrics should a multi-agent knowledge-architecture system track? Beyond token cost and latency — what about research quality, evidence freshness, scaffold coverage, method improvement rate, and institutional maturity progression?

15. How do production systems detect drift — when an agent's behavior gradually diverges from its configured intent? What drift-detection patterns exist in the harness engineering ecosystem?

16. What intervention models exist for "the system is doing something weird"? How do production orchestrators surface anomalies, trigger human review, and halt operations?

17. How do existing systems handle cost tracking and budget enforcement across heterogeneous multi-model agent fleets? What patterns exist for attributing costs to specific research programs or production cycles?

**Wave 4 — Lane E: Trust and Security (5 questions)**

18. What trust/auth models exist for multi-agent systems where agents have different permission levels? Examine OpenClaw's permission manifests, Symphony's SPEC.md security model, AAIF's Agentgateway (enterprise MCP security), and A2A's security cards.

19. What attack vectors are specific to multi-agent orchestration systems? Beyond prompt injection — what about inter-agent manipulation, allowlist bypass (OpenClaw's `time` wrapper attack), cross-agent state poisoning, and unauthorized escalation?

20. How do production systems implement write-scope governance — controlling which agents can modify which parts of a shared repository? Examine Git branch protection, Claude Code hooks (PreToolUse for write filtering), and Hermes Agent's skill permission model.

21. What rollback/reversibility patterns exist for agent-produced artifacts? How do systems undo an agent's changes when they fail review? Beyond git revert — what about state rollback, memory rollback, and cross-surface consistency after partial rollback?

22. How do the AAIF security standards (Agentgateway, A2A security cards) interact with individual harness security models (Claude Code permissions, Codex sandbox, OpenClaw exec approvals)? Is there a unified trust model or do they remain layered?

**Wave 5 — Open Research Slots (6 questions)**

23. What is the structural taxonomy of agent memory architectures as of March 2026? Classify: episodic (Graphiti, Zep), semantic (Pinecone, Qdrant, Weaviate), procedural (Hermes Agent skills), knowledge-graph (GraphRAG, Cognee, FalkorDB), and user-model (Honcho, Mem0). How do these map to a multi-agent system where different agents need different memory types?

24. What coordination/event choreography primitives exist in the Make/Zapier/webhook ecosystem that are relevant to AI agent orchestration? How do existing automation platforms handle the trigger→action→verify pattern that agent orchestration needs?

25. What is the state of the art in cross-agent memory — agents sharing knowledge within a multi-harness environment? Examine ACP's state-passing, Hermes Agent's FTS5 cross-session search, OpenClaw's supermemory, and any emerging shared-memory protocols.

26. What backend/cloud substrate patterns are emerging for always-on agent infrastructure? Examine Daytona (serverless persistence), Modal (serverless GPU), E2B (secure sandboxing), and the Perplexity Personal Computer pattern (always-on Mac mini). What is the minimal viable infrastructure for a persistent multi-agent system?

27. What self-improving agent patterns exist beyond Hermes Agent's skill-creation loop? Examine DeerFlow's cross-session user profiling, OpenViking's three-tier context loading, AutoMemory's habit learning, and any RL-from-agent-experience systems.

28. What is the current state of the AGENTS.md standard's adoption and how is it evolving? What are the best practices for writing AGENTS.md files that the research shows actually improve agent performance vs the finding that LLM-generated AGENTS.md files reduce performance?

**Wave 6 — Integration and Telos (4 questions)**

30. What is the build order for the five Phase 3 lanes given dependency analysis? Which lane's artifacts are prerequisites for which other lanes? What can be parallelized?

Where are the grammar layer's blind spots relative to the production evidence?
