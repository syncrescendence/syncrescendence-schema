## What Was Discovered This Session

### The Repo as Constitutional Substrate

Both Claude.ai and ChatGPT have native GitHub connectors providing read access to repositories in conversation. Claude.ai can sync repo files as Project Knowledge (searchable, refreshable). ChatGPT reads repos via Deep Research/Agent modes. A single GitHub repo can serve as the shared source of truth between both chat surfaces and the full CLI fleet (Claude Code, Codex, Gemini CLI — all with read-write via GitHub MCP server).

### The Inverted Relay

The Schema-3 operating model is being restructured. Previously: Sovereign handcrafted the mystical recipe (steering prompts for Vanguard), Vanguard generated structured commissions, Sovereign relayed to Vizier, Vizier produced artifacts. The handcrafted steering was effortful and fickle. The new model: Vizier crafts model-specific commissions for each avatar (Vanguard, Oracle, Cartographer, Adjudicator), the Sovereign pastes with minimal steering, results return to Vizier for synthesis. The mystical recipe becomes Vizier's responsibility — encoding each model's cognitive signature into the commission structure.

### The Connector Reliability Problem (Oracle Intelligence)

Grok's adversarial research confirmed: GitHub connectors on all chat surfaces are unreliable. Claude.ai sync breaks (files not indexed despite "Connected" status). ChatGPT gates access behind specific modes with 5-10 minute indexing delays. Gemini takes frozen snapshots with no push reflection. **The Oracle's key finding: "one repo + MCP memory layer" is the actual architecture.** MCP provides the stable substrate; GitHub connectors supplement when they work.
