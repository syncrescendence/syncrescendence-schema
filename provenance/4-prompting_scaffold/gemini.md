### The Bifurcation Logic

Your examples made the case better than any research could. Example B — vanilla 3.1 Pro — produced "Ralph-Wiggum reset," "drowning in its own exhaust," and genuinely clearer mechanistic explanation. The bifurcation maps each surface to the capability profile it can actually sustain. The web app gets the **explainer** role — Gemini's sweet spot of translating density into clarity at an advanced register — using lightweight behavioral snippets that shape output without triggering persona performance. AI Studio gets the **shape rotator / synthesis engine** role — cross-disciplinary bridging, novel hypothesis generation, pan-science integration — using the full XML architecture that the unlimited system instruction field can absorb without collapsing into templates.

### Web App: Why Five Light Snippets

Second, the research confirms that overloaded prompts on Gemini's web app cause template collapse — the model over-parses dense instruction blocks and defaults to rigid formatting. Third, the snippets function partly as memory in Gemini's architecture, which means they persist differently than a conventional system prompt: each one is a semi-independent behavioral nudge rather than a hierarchically parsed directive set.

Each snippet targets one axis of behavior, which means you can add, remove, or swap individual snippets without cascading effects. If Gemini starts defaulting to bullet lists after a long thread, the voice snippet is doing that work independently. If it starts hedging, the forbids snippet (Snippet 5) can be deployed as reinforcement without touching the others.
