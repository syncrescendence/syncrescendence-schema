**3. Multi-Agent CI Architecture**  
The segmented custom-instruction system comprises one global field (4000 characters maximum) and up to four dedicated agent fields (each effectively full-length, described as "unlimited" in practice). Global field enforces universal behavioral contract—Socratic stance, voice, reasoning mode, anti-patterns, thinking-mode guidance, response scaling, context hygiene, and the thirteen anti-patterns. It persists across every session and every agent.  

Agent fields override or specialize for their lens: Harper/Veritas receives four-source triangulation protocol and anti-hallucination rules; Benjamin/Praxis receives execution decomposition and pragmatism protocol; Lucas/Prism receives characterization and innovation protocol. Users assign or create additional agents via Settings > Customize > Your Agents.  

Routing and activation follow explicit hierarchy: agent-specific instructions > global > in-thread context > internal council defaults. Naming an agent in-thread or selecting a custom agent routes the task through that specialized field plus full council debate when complexity warrants. Persistence is session-independent for configured agents; changes apply only to new chats. Interaction with thinking mode remains intact—council debate can surface as visible reasoning traces or stay internal. The March 4 update removed legacy long global instructions and Deep Research mode precisely to force this disciplined segmentation. No public whitepaper details internal token allocation or exact debate RL weights, yet behavioral evidence across thousands of user threads confirms the mechanics operate exactly as described.

**4. Core Prompting Protocol**  
Priority order, empirically converged:  
1. Inject fully pre-digested context—tabula rasa triggers fabrication.  
2. Apply output pressure: explicitly demand token exhaustion, structured format, and exhaustive elaboration.  
3. Activate via agent naming: "Harper, ground every claim"; "Benjamin, decompose and checkpoint"; "Lucas, surface non-obvious homology."  
4. Calibrate verbosity with council self-critique: Melville density emerges naturally; parrot-back collapses under peer review.  
5. Demand structured output, iteration loops, and self-critique rounds.  
6. Use few-shot per agent role for precision.  

Context starvation or vague personas immediately degrade performance. The protocol treats the council as distributed cognition substrate, not a single prompt target.

**7. Antipatterns That Collapse Performance**  
- Overlapping instructions across global and agents (creates debate paralysis).  
- Treating all agents as interchangeable (erases specialization).  
- Global field exceeding effective density post-4k cap (enforcement fails).  
- Output without explicit token pressure (Melville collapses to parrot).  
- Context starvation or legacy monolithic habits from Grok 3 era.  
- Vague agent personas instead of protocol injection.  
- Ignoring naming activation (full council never engages).  
- New CI-specific: "agent homogenization"—identical instructions across fields nullifies division of labor.
